# Agent Lab 90 秒演示脚本

> 口播与屏幕操作逐段对应；数字引用 2026-07-18 真实任务 #16，不使用未验证的提升结论。

## 0～10 秒：项目定位与自然语言目标

- 操作：打开 `/agent-lab/`，在策略目标输入框预填「尽量扩大可活动区域，避免进入狭窄通道」。
- 屏幕：同时展示自然语言目标和最大迭代选项，不提交任务。
- 口播：这是 KOB Agent Lab。我只需描述「尽量扩大活动区域、避免狭窄通道」，Agent 就会自主生成、编译、评测并改进 Java Bot，最终交付最佳版本。

## 10～25 秒：创建任务与 Workflow 时间线

- 操作：沿用已填写的目标，选 3 轮，点「开始进化」。
- 屏幕：时间线依次点亮 生成代码 → 沙箱编译 → 公开评测 → 失败分析 → 策略改进 → 隐藏验证。
- 口播：Agent 在受控工具集内闭环，每秒刷新进度，状态机用乐观锁单向推进。

## 25～45 秒：V1～V3 指标与失败原因

- 操作：在版本对比表依次指向 V1～V3，再展开当次任务中被拒版本或失败 Trace 的摘要。
- 屏幕：展示任务 #16 的 V1～V3：公开集各 48 局、得分均为 0.0104、合法移动率 100%，V3 标记为最佳；本次无失败版本。
- 口播：每个版本先跑 48 局公开集，再由 Agent 根据真实结果改进下一版；指标全部由确定性引擎计算，不靠模型自评。本次三版都合法，但公开得分没有提升，我不会补造成功故事。

## 45～60 秒：Trace、Token、幂等与恢复

- 屏幕：Trace 表展示阶段、工具、耗时、Token 与脱敏摘要；V1～V3 的 LLM Token 分别为 2197、4809、11577。
- 口播：每次工具调用带幂等键，重启后从最后成功 Step 续跑；Token 与延迟可观测。Token 增长也提醒我们，代码上下文变长会直接增加成本。

## 60～75 秒：隐藏验证与最佳版本

- 屏幕：任务完成后展示每版 24 局隐藏验证，三个版本得分均为 0，V3 在同分规则下成为唯一最佳版本。
- 口播：隐藏集只在迭代结束后运行，结果不反馈给模型。这次没有证明策略提升，所以我只展示系统如何阻止模型自评冒充真实改进。

## 75～85 秒：保存 Bot 与播放 Replay

- 操作：点「保存为我的 Bot」，再播放一条代表性录像。
- 屏幕：版本保存为正式 Bot；Replay 面板复用对战棋盘播放任务 #16 的代表性失败对局。
- 口播：最佳版本一键存为我的 Bot，代表性失败录像可回放复盘。

## 85～90 秒：安全边界与生产化方向

- 操作：切到 README 的「沙箱与安全边界」架构说明，指向独立 Bot 子进程和生产容器化方向。
- 屏幕：高亮「禁网络与文件写」以及「容器、cgroup、seccomp」两组关键词。
- 口播：候选代码在独立沙箱子进程运行，禁网络与文件写；当前 SecurityManager 适用于本地演示，生产应迁移到容器与 cgroup。
